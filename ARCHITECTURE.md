# MES 生产执行系统版本自动化发布运维平台 - 系统架构设计

## 一、整体架构概览

### 1.1 系统分层架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     门户展示层 (Portal Layer)                           │
│  发布申请入口 │ 审批工作台 │ 监控大屏 │ 报表中心 │ 运维操作平台        │
├─────────────────────────────────────────────────────────────────────────┤
│                     业务服务层 (Service Layer)                          │
│  发布管理服务 │ 审批流程引擎 │ 监控告警服务 │ 智能回滚决策引擎        │
│  报表分析服务 │ 应急演练服务 │ 通知推送服务 │ 审计追踪服务            │
├─────────────────────────────────────────────────────────────────────────┤
│                     核心业务层 (Core Domain Layer)                     │
│  前置校验引擎 │ 灰度部署引擎 │ 故障根因分析引擎 │ 数据兜底保障引擎    │
├─────────────────────────────────────────────────────────────────────────┤
│                     数据访问层 (Data Access Layer)                     │
│  配置管理中心 │ 数据持久化 │ 分布式缓存 │ 外部系统适配器              │
├─────────────────────────────────────────────────────────────────────────┤
│                     基础设施层 (Infrastructure Layer)                 │
│  定时任务调度器 │ 消息队列中间件 │ 日志采集器 │ 监控数据探针          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 核心业务流程全景

#### 流程1：版本发布全生命周期
```
研发提交申请 → 前置条件校验 → 风险等级评估 → 分级审批流程 
    → 灰度策略生成 → 分批部署执行 → 实时监控观察 → 全量发布确认
```

#### 流程2：智能回滚应急处置
```
监控指标异常 → 阈值触发判定 → 自动版本回滚 → 产线权限锁定
    → 故障根因分析 → 影响范围评估 → 多渠道通知推送 → 修复验证恢复
```

#### 流程3：应急演练执行
```
手动发起演练 → 演练方案生成 → 模拟故障注入 → 自动化回退执行
    → 人工兜底演练 → 过程完整记录 → 整改建议输出 → 演练报告归档
```

#### 流程4：运维报表分析
```
定时任务触发 → 多维度数据统计 → 趋势图表生成 → PDF/Excel报告输出
    → 多渠道分发 → 历史数据归档
```

## 二、核心模块职责划分

### 2.1 前置校验模块 (PreCheck Module)
- **自动化测试覆盖率校验**：对接 Jenkins/Allure，校验单元测试、集成测试覆盖率达标
- **代码安全合规审查**：对接 SonarQube，校验漏洞等级、代码异味、重复率
- **PLC 设备接口探测**：工业协议握手 (OPC UA/Modbus)，测试数据读写连通性
- **WMS 依赖系统健康检查**：REST API 健康探测、消息队列连通性验证

### 2.2 审批流程模块 (Approval Module)
- **风险等级识别**：
  - L1 常规版本：生产流程优化、功能迭代
  - L2 紧急版本：数据漏洞修复、安全补丁
- **分级审批矩阵**：
  | 风险等级 | 生产负责人 | 质量经理 | 运维审批人 | 工厂总监 |
  |---------|-----------|---------|-----------|---------|
  | L1 常规 | ✅ 审批   | ✅ 审批 | ✅ 审批   | ❌       |
  | L2 紧急 | ✅ 审批   | ✅ 审批 | ✅ 审批   | ✅ 审批 |

### 2.3 灰度部署模块 (Deployment Module)
- **车间产线灰度策略**：
  - 阶段1：试点产线 (10% 流量)
  - 阶段2：扩展产线 (30% 流量)  
  - 阶段3：半量产线 (50% 流量)
  - 阶段4：全量产线 (100% 流量)
- **版本管理**：版本仓库、版本校验、版本回滚点快照

### 2.4 监控回滚模块 (Monitor Module)
- **核心监控指标**：
  - 工单上报报错率 (阈值: >2%)
  - 设备数据采集响应延迟 (阈值: >500ms)
  - 生产工艺参数异常次数 (阈值: >5次/5min)
- **回滚决策引擎**：多指标加权评分，超过阈值自动触发回滚

### 2.5 故障分析模块 (Analysis Module)
- **影响范围评估**：受影响车间、产线、设备清单
- **不良品预估**：基于生产速率、停机时长、历史不良率计算
- **根因分析**：基于日志、指标、异常堆栈的关联分析

### 2.6 审计日志模块 (Audit Module)
- **操作类型全覆盖**：部署、回滚、权限变更、告警、审批
- **日志字段**：操作人、时间、IP、操作类型、请求参数、响应结果、耗时
- **不可篡改**：哈希链式存储，定期归档

## 三、产线停机故障数据兜底方案

### 3.1 多级数据缓存架构

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  PLC 实时数据   │───▶│ 边缘计算网关    │───▶│ MES 内存缓存    │
│  (Level 0)      │    │  (Level 1)      │    │  (Level 2)      │
└─────────────────┘    └─────────────────┘    └─────────────────┘
                                 │                    │
                                 ▼                    ▼
                        ┌─────────────────┐    ┌─────────────────┐
                        │ 本地持久化文件  │    │  分布式数据库   │
                        │  (Level 3)      │    │  (Level 4)      │
                        └─────────────────┘    └─────────────────┘
```

### 3.2 故障兜底机制

#### 场景1：MES 系统崩溃，PLC 数据无法上报
- **兜底方案**：边缘网关自动切换到本地缓存模式，数据写入本地环形缓冲区
- **数据完整性**：支持 8 小时本地缓存，断电不丢失
- **恢复机制**：MES 恢复后自动断点续传，数据完整性校验

#### 场景2：数据库宕机，无法写入生产数据
- **兜底方案**：切换到本地 SQLite 应急数据库，保证核心生产数据不丢
- **同步机制**：数据库恢复后自动异步同步，冲突自动合并

#### 场景3：网络中断，车间与总部失联
- **兜底方案**：车间级独立运行模式，本地完成生产调度
- **数据同步**：网络恢复后自动双向同步，确保数据一致性

### 3.3 人工线下兜底流程

1. **系统故障触发**：自动回滚失败或系统完全不可用
2. **告警通知**：声光报警器 + 短信 + 电话通知运维团队
3. **人工接管**：生产负责人确认后，切换到线下手工模式
4. **数据记录**：使用应急纸质单据 + 本地 Excel 模板记录生产数据
5. **恢复补录**：系统恢复后 2 小时内完成数据补录与校验

## 四、技术栈选型

| 层级 | 技术选型 |
|-----|---------|
| 开发语言 | Python 3.10+ |
| 定时任务 | APScheduler |
| 数据持久化 | SQLite (开发) / PostgreSQL (生产) |
| 分布式缓存 | Redis |
| 报表生成 | ReportLab (PDF) / OpenPyXL (Excel) / Matplotlib (图表) |
| 日志框架 | Python Logging + RotatingFileHandler |
| 配置管理 | YAML + Pydantic |
| 外部系统对接 | requests (HTTP) / opcua (工业协议) |
| 消息通知 | SMTP (邮件) / WebHook (企业微信/钉钉) |
